# 投资资产管理系统说明文档

> 更新时间：2026-05-19 06:45 CST  
> 适用项目：本地运行的 `invest-tracker` 投资资产管理系统

---

## 1. 系统定位

本系统用于管理个人投资组合与资产配置，覆盖：

- A股权益、A股ETF、港股ETF、债券基金、REITs、黄金；
- 证券现金、银行存款、申购在途资产；
- 持仓明细、资产配置、资产快照、银行存款、交易录入/管理、现金设置；
- 交易记录、证券资金流水、持仓校正、成本口径、手续费估算、导入导出。

当前版本是本地单用户 MVP：

- 前端：Vue 3 + Element Plus + ECharts，单文件 `frontend/index.html`；
- 后端：FastAPI；
- 数据库：SQLite，默认路径 `data/invest.db`；
- 部署方式：前端与后端都可通过 `docker compose` 启动；前端由 nginx 提供静态页面，并通过统一的 `/api` 反向代理转发到后端；
- 后端数据库路径支持宿主机与容器环境，默认都会落到项目目录下的 `data/invest.db`。

## 2. 运行方式

项目路径：

```bash
/Users/jian/invest-tracker
```

推荐启动方式：

```bash
cd /Users/jian/invest-tracker
docker compose up -d --build
```

如只需单独重启前端或后端：

```bash
cd /Users/jian/invest-tracker
docker compose up -d --build frontend
docker compose up -d --build backend
```

访问地址：

- 前端页面：http://localhost:8080
- 前端代理 API：http://localhost:8080/api
- 后端 API（宿主机直连）：http://localhost:8000
- SQLite 数据库：`/Users/jian/invest-tracker/data/invest.db`

说明：

- 前端统一通过 `/api` 访问后端，不再在页面代码中写死 `8000/8001` 端口；
- `frontend/index.html` 和 `frontend/nginx.conf` 通过 bind mount 挂载到 nginx 容器，修改后通常重启前端容器或强刷浏览器即可生效；
- 后端代码修改后需要重建/重启 backend 容器；
- 后端数据库路径默认兼容宿主机和容器环境，未显式设置 `DB_PATH` 时使用项目目录下的 `data/invest.db`。

## 3. 当前主要功能

### 3.1 顶部资产总览

展示：

- 总资产；
- 投资账户市值；
- 证券现金；
- 银行存款；
- 投资账户盈亏；
- 申购在途。

配色遵循中国市场习惯：盈利红色，亏损绿色。

### 3.2 资产快照

默认首页，用于记录和分析每日资产状态。

功能：

- 记录/更新今日快照；
- 同日快照自动更新，不重复新增；
- “今日”按后端本地时区计算，默认使用 `Asia/Shanghai`；
- 可通过环境变量 `APP_TIMEZONE` 调整后端时区；
- 总资产趋势图；
- 当前资产结构图；
- 区间变化明细；
- 快照历史列表。

快照字段包括：日期、总资产、投资账户市值、银行存款、证券现金、申购在途、投资账户盈亏、投资占比、现金+存款+在途、持仓数。

### 3.3 资产配置

用于分析组合结构、风险暴露和预计收益。

包含：

- 权益资产占比；
- 固收 + 存款占比；
- 组合预计年化；
- 当前申购在途；
- 权益 / 固收 / 存款大类结构图；
- 细分类别占比图；
- 配置健康检查；
- 大类汇总表和细分类别明细表。

当前大类口径：

| 大类 | 包含 |
|---|---|
| 权益 | A股权益、A股ETF、港股ETF、REITs、黄金 |
| 固收 | 债基、证券现金、基金申购在途 |
| 存款 | 银行存款 |

组合预计年化：

```text
组合预计年化 = Σ(资产金额 × 预计年化收益率) / Σ资产金额
```

### 3.4 持仓明细

展示当前所有确认持仓。

主要字段：

- 名称、分类、代码；
- 持仓数量；
- 普通成本；
- 摊薄成本；
- 最新价；
- 市值；
- 累计盈亏；
- 收益率；
- 预计年化收益；
- 近一年标的收益率。

操作：

- 编辑预计年化收益；
- 持仓强制校正；
- 查看校正记录；
- 点击持仓行查看该标的交易记录；
- 点击「同步最新价」更新最新价；
- 点击「同步近一年收益率」更新近一年标的收益率。

说明：`申购待确认` 不生成正式持仓，不参与持仓盈亏。

### 3.5 收益分析

用于评估组合的真实投资收益。

核心指标：

| 指标 | 说明 |
|---|---|
| 当前总资产 | 持仓市值 + 证券现金 + 银行存款 + 申购在途 |
| 累计净投入 | 所有「投入」流水合计 - 所有「取出」流水合计 |
| 累计总收益 | 当前总资产 - 累计净投入 |
| XIRR 年化 | 按外部现金流 + 当前总资产计算的资金加权年化收益率 |
| 浮盈 + 分红 | 持仓浮动盈亏 + 累计分红合计 |
| YTD 收益 | 年初至今收益，基于年初快照调整 |

图表：

- 资产 vs 净投入趋势图；
- 标的收益贡献表（按贡献排序）。

组合资金流水：

- 记录「组合外部」资金进出（新增投入、提取离场）；
- 不同于银证转入/转出（那是组合内部搬家）；
- 买入/卖出/分红是内部资产转换，不在此处记录。

首次使用建议：先录入历史累计投入作为初始基准。

### 3.6 银行存款

用于维护和分析银行存款。

功能：

- 新增、编辑、删除存款；
- 存款总额；
- 存款占总资产比例；
- 加权平均利率；
- 预计年利息；
- 下一笔到期；
- 银行集中度；
- 到期分布；
- 下载存款 CSV 模板；
- 导出存款 CSV；
- 导入存款 CSV。

存款导入模板字段：

```text
bank_name, amount, interest_rate, due_date, remark
```

导入说明：

- 当前支持 `.csv`；
- `due_date` 日期格式为 `YYYY-MM-DD`，可为空；
- 导入前系统自动备份 `data/invest.db` 到 `backups/`；
- 成功行写入真实存款表，失败行会在导入结果中提示。

### 3.6 交易录入/管理

交易录入和交易管理合并在同一页面。

支持方向：

- 买入；
- 卖出；
- 分红；
- 申购待确认。

交易字段：

```text
日期、证券账户、代码、名称、分类、方向、数量、单价、金额、手续费、备注
```

自动匹配规则：

- 输入已有持仓代码时，精确匹配名称和分类；
- 输入已有持仓名称时，精确匹配或唯一候选时自动匹配；
- 新标的可直接录入，不会被旧持仓错误覆盖；
- 新标的按代码/名称推断分类；
- 分类可手动修改；
- 不自动填交易价格，价格必须按真实成交价手动录入；
- 手续费按所选证券账户和资产分类自动估算，但可手动覆盖。

交易管理支持：

- 按日期范围、代码、名称、方向筛选；
- 编辑交易；
- 删除交易；
- 查看申购待确认记录；
- 下载交易 CSV 模板；
- 导出交易 CSV；
- 导入交易 CSV。

交易导入模板字段：

```text
date, account, code, name, category, direction, quantity, price, amount, fee, remark
```

导入说明：

- 当前支持 `.csv`；
- `date` 日期格式为 `YYYY-MM-DD`；
- `direction` 必须是 `买入`、`卖出`、`分红`、`申购待确认`；
- `amount` 填正数，买入/卖出方向由 `direction` 表示；
- 导入前系统自动备份数据库；
- 导入成功后自动重算持仓、普通成本、摊薄成本、证券现金和申购在途。

### 3.7 现金设置

用于校准证券账户现金余额、记录银证资金进出、维护交易费率。

包含：

- 当前证券现金自动余额；
- 手动校准余额；
- 证券资金流水；
- 资金流水查询；
- 区间转入、区间转出、区间净额、当前证券现金汇总；
- 多证券账户交易费率设置。

证券现金口径：

```text
证券现金 = 现金基准 + 资金流水净额 + 交易现金流
```

交易现金流规则：

| 方向 | 现金流 |
|---|---|
| 买入 | `-(金额 + 手续费)` |
| 申购待确认 | `-(金额 + 手续费)` |
| 卖出 | `金额 - 手续费` |
| 分红 | `金额 - 手续费` |

注意：买入、卖出、分红、申购待确认应记录在「交易录入/管理」；银证转入、银证转出、现金校准、其他调整应记录在「证券资金流水」。

### 3.8 证券资金流水

非交易类资金变化通过 `cash_flows` 记录。

支持类型：

| 类型 | 说明 | 对证券现金影响 |
|---|---|---:|
| 银证转入 | 银行转入证券账户 | 增加 |
| 银证转出 | 证券账户转出到银行 | 减少 |
| 现金校准 | 按券商实际余额修正差额 | 按差额增减 |
| 其他调整 | 手工修正 | 按输入金额增减 |

支持按日期、证券账户、流水类型查询，支持新增、编辑、删除。

### 3.9 多证券账户费率与手续费估算

现金设置页提供「交易费率设置」。

支持：

- 多个证券账户分别保存费率；
- 每个账户按资产类别维护佣金率、印花税率、过户费率、最低佣金；
- 交易录入选择证券账户后自动估算手续费；
- 手续费输入框可手动覆盖；
- 手动覆盖后，后续修改金额/分类/方向/账户不会自动覆盖手动值。

默认口径：

- A股权益：佣金、卖出印花税、过户费；
- ETF / REITs / 黄金：默认只按佣金估算；
- 债基：默认 0；
- 费率单位是 `%`，例如万2.5填写 `0.025`。

华泰普通账户示例：

| 分类 | 佣金率 | 印花税 | 过户费 | 最低佣金 |
|---|---:|---:|---:|---:|
| A股权益 | 0.013 | 0.05 | 0.001 | 5 |
| A股ETF / 港股ETF / REITs / 黄金 | 0.01 | 0 | 0 | 5 |
| 债基 | 0 | 0 | 0 | 0 |

### 3.10 持仓强制校正

用于解决券商实际持仓数量/成本价与系统按交易记录重算结果不一致的问题。

规则：

- 校正记录是计算锚点，不修改历史交易；
- 无校正时：所有交易记录参与持仓重算；
- 有校正时：以最新校正锚点为基础，只叠加校正日期之后的交易；
- 校正记录可查询、可删除；
- 删除校正记录后系统重新按剩余记录计算。

适用场景：

- 历史交易不完整；
- 早期手续费、分红、拆分等记录缺失；
- 券商成本口径与系统普通成本不一致；
- 需要以券商当前显示为准重新锚定。

### 3.11 普通成本与摊薄成本

系统保留两个成本口径：

```text
普通成本价 = 剩余持仓普通成本 / 剩余持仓数量
券商摊薄成本 = (累计买入现金 - 累计卖出回款 - 累计分红) / 剩余持仓数量
```

说明：

- 普通成本用于计算剩余确认持仓的浮动盈亏；
- 券商摊薄成本尽量贴近券商 APP 的持仓成本口径；
- 当历史卖出回款 + 分红超过累计买入现金时，摊薄成本可以为负数；
- 累计盈亏仍按普通成本口径计算，避免负成本导致重复计算收益。

农业银行示例：

```text
(累计买入 288,018.19 - 卖出回款 330,137.19 - 累计分红 43,934.21) / 41,400 ≈ -2.0786
```

### 3.12 近一年标的收益率

持仓明细新增「近一年收益」列。

口径：

```text
近一年标的收益率 = 标的自身过去一年价格/净值变化率
```

说明：

- 它是标的历史表现，不是账户实际持有收益；
- 不考虑你的买入/卖出时点、仓位变化、成本价、摊薄成本和资金流；
- 可与「预计年化收益」对比：预计年化是未来假设，近一年收益是过去表现；
- A股 / ETF / REIT / 黄金优先使用前复权 K 线；
- 场外基金使用天天基金累计净值，必要时回退到单位净值；
- 外部行情源缺失或失败时显示「暂无数据」。

## 4. 核心数据口径

### 4.1 总资产

```text
总资产 = 投资账户市值 + 证券现金 + 银行存款 + 申购在途
```

### 4.2 投资账户市值

```text
投资账户市值 = Σ(确认持仓数量 × 最新价)
```

### 4.3 投资账户盈亏

```text
投资账户盈亏 = Σ[(最新价 - 普通成本价) × 持仓数量 + 累计分红]
```

说明：申购在途不计入投资账户盈亏。

### 4.4 收益率

```text
收益率 = 累计盈亏 / 普通持仓成本
```

### 4.5 组合预计年化

```text
组合预计年化 = Σ(资产金额 × 预计年化收益率) / Σ资产金额
```

### 4.6 申购在途

适用场景：场外基金申购后，券商现金已扣减或冻结，但份额和确认净值尚未给出。

```text
申购在途 = Σ(方向为 申购待确认 的 金额 + 手续费)
```

规则：

- 计入总资产；
- 计入资产配置中的固收；
- 不计入正式持仓数量；
- 不计入持仓成本和盈亏；
- 份额确认后，编辑原交易为 `买入`，补充实际份额和确认净值。

## 5. 数据表

| 表 | 用途 |
|---|---|
| `holdings` | 当前确认持仓，由交易记录和校正锚点重算 |
| `transactions` | 交易记录，含证券账户字段 |
| `holding_corrections` | 持仓强制校正 / 对账锚点 |
| `cash_flows` | 证券资金流水：银证转入、银证转出、现金校准、其他调整 |
| `deposits` | 银行存款 |
| `settings` | 系统设置，如证券现金基准、交易费率配置 |
| `daily_snapshots` | 每日资产快照 |

### 5.1 `holdings`

主要字段：

```text
code, name, category, quantity, avg_cost, diluted_cost, total_dividend,
last_price, expected_return, trailing_return_1y,
trailing_return_1y_source, trailing_return_1y_updated_at, updated_at
```

### 5.2 `transactions`

主要字段：

```text
date, code, name, category, account, direction, quantity, price, amount, fee, remark
```

### 5.3 `holding_corrections`

主要字段：

```text
date, code, name, category, actual_quantity, actual_avg_cost,
actual_total_cost, actual_total_dividend, remark, created_at
```

### 5.4 `cash_flows`

主要字段：

```text
date, account, flow_type, amount, balance_before, balance_after, remark, created_at
```

### 5.5 `daily_snapshots`

主要字段：

```text
date, total_assets, total_market_value, bank_balance, securities_cash,
pending_purchase, total_profit, holdings_count, created_at
```

### 5.6 `portfolio_cash_flows`

主要字段：

```text
date, flow_type（投入/取出）, amount, source, remark, created_at
```

用途：记录组合级外部资金进出，用于计算 XIRR 年化收益。不同于银证转入/转出（组合内部）和买卖交易（资产转换）。
