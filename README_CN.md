# 投资资产管理系统说明文档

> 更新时间：2026-05-13 08:45 CST  
> 适用项目：本地运行的 `invest-tracker` 投资资产管理系统

---

## 1. 系统定位

本系统用于管理个人投资组合与资产配置，覆盖：

- A股权益、A股ETF、港股ETF、债券基金、REITs、黄金；
- 证券现金、银行存款、申购在途资产；
- 持仓明细、资产配置、资产快照、银行存款、交易录入/管理、现金设置；
- 交易记录、证券资金流水、持仓校正、成本口径、手续费估算、导入导出。

当前版本是本地单用户 MVP：

- 前端：Vue 3 + Element Plus + ECharts，单文件 `frontend/index.html`；
- 后端：FastAPI；
- 数据库：SQLite，路径 `data/invest.db`；
- 前端通过 Docker nginx 提供静态页面，后端直接在主机运行。

## 3. 当前主要功能

### 3.1 顶部资产总览

展示：

- 总资产；
- 投资账户市值；
- 证券现金；
- 银行存款；
- 投资账户盈亏；
- 申购在途。

配色遵循中国市场习惯：盈利红色，亏损绿色。

### 3.2 资产快照

默认首页，用于记录和分析每日资产状态。

功能：

- 记录/更新今日快照；
- 同日快照自动更新，不重复新增；
- 总资产趋势图；
- 当前资产结构图；
- 区间变化明细；
- 快照历史列表。

快照字段包括：日期、总资产、投资账户市值、银行存款、证券现金、申购在途、投资账户盈亏、投资占比、现金+存款+在途、持仓数。

### 3.3 资产配置

用于分析组合结构、风险暴露和预计收益。

包含：

- 权益资产占比；
- 固收 + 存款占比；
- 组合预计年化；
- 当前申购在途；
- 权益 / 固收 / 存款大类结构图；
- 细分类别占比图；
- 配置健康检查；
- 大类汇总表和细分类别明细表。

当前大类口径：

| 大类 | 包含 |
|---|---|
| 权益 | A股权益、A股ETF、港股ETF、REITs、黄金 |
| 固收 | 债基、证券现金、基金申购在途 |
| 存款 | 银行存款 |

组合预计年化：

```text
组合预计年化 = Σ(资产金额 × 预计年化收益率) / Σ资产金额
```

### 3.4 持仓明细

展示当前所有确认持仓。

主要字段：

- 名称、分类、代码；
- 持仓数量；
- 普通成本；
- 摊薄成本；
- 最新价；
- 市值；
- 累计盈亏；
- 收益率；
- 预计年化收益；
- 近一年标的收益率。

操作：

- 编辑预计年化收益；
- 持仓强制校正；
- 查看校正记录；
- 点击持仓行查看该标的交易记录；
- 点击「同步最新价」更新最新价；
- 点击「同步近一年收益率」更新近一年标的收益率。

说明：`申购待确认` 不生成正式持仓，不参与持仓盈亏。

### 3.5 银行存款

用于维护和分析银行存款。

功能：

- 新增、编辑、删除存款；
- 存款总额；
- 存款占总资产比例；
- 加权平均利率；
- 预计年利息；
- 下一笔到期；
- 银行集中度；
- 到期分布；
- 下载存款 CSV 模板；
- 导出存款 CSV；
- 导入存款 CSV。

存款导入模板字段：

```text
bank_name, amount, interest_rate, due_date, remark
```

导入说明：

- 当前支持 `.csv`；
- `due_date` 日期格式为 `YYYY-MM-DD`，可为空；
- 导入前系统自动备份 `data/invest.db` 到 `backups/`；
- 成功行写入真实存款表，失败行会在导入结果中提示。

### 3.6 交易录入/管理

交易录入和交易管理合并在同一页面。

支持方向：

- 买入；
- 卖出；
- 分红；
- 申购待确认。

交易字段：

```text
日期、证券账户、代码、名称、分类、方向、数量、单价、金额、手续费、备注
```

自动匹配规则：

- 输入已有持仓代码时，精确匹配名称和分类；
- 输入已有持仓名称时，精确匹配或唯一候选时自动匹配；
- 新标的可直接录入，不会被旧持仓错误覆盖；
- 新标的按代码/名称推断分类；
- 分类可手动修改；
- 不自动填交易价格，价格必须按真实成交价手动录入；
- 手续费按所选证券账户和资产分类自动估算，但可手动覆盖。

交易管理支持：

- 按日期范围、代码、名称、方向筛选；
- 编辑交易；
- 删除交易；
- 查看申购待确认记录；
- 下载交易 CSV 模板；
- 导出交易 CSV；
- 导入交易 CSV。

交易导入模板字段：

```text
date, account, code, name, category, direction, quantity, price, amount, fee, remark
```

导入说明：

- 当前支持 `.csv`；
- `date` 日期格式为 `YYYY-MM-DD`；
- `direction` 必须是 `买入`、`卖出`、`分红`、`申购待确认`；
- `amount` 填正数，买入/卖出方向由 `direction` 表示；
- 导入前系统自动备份数据库；
- 导入成功后自动重算持仓、普通成本、摊薄成本、证券现金和申购在途。

### 3.7 现金设置

用于校准证券账户现金余额、记录银证资金进出、维护交易费率。

包含：

- 当前证券现金自动余额；
- 手动校准余额；
- 证券资金流水；
- 资金流水查询；
- 区间转入、区间转出、区间净额、当前证券现金汇总；
- 多证券账户交易费率设置。

证券现金口径：

```text
证券现金 = 现金基准 + 资金流水净额 + 交易现金流
```

交易现金流规则：

| 方向 | 现金流 |
|---|---|
| 买入 | `-(金额 + 手续费)` |
| 申购待确认 | `-(金额 + 手续费)` |
| 卖出 | `金额 - 手续费` |
| 分红 | `金额 - 手续费` |

注意：买入、卖出、分红、申购待确认应记录在「交易录入/管理」；银证转入、银证转出、现金校准、其他调整应记录在「证券资金流水」。

### 3.8 证券资金流水

非交易类资金变化通过 `cash_flows` 记录。

支持类型：

| 类型 | 说明 | 对证券现金影响 |
|---|---|---:|
| 银证转入 | 银行转入证券账户 | 增加 |
| 银证转出 | 证券账户转出到银行 | 减少 |
| 现金校准 | 按券商实际余额修正差额 | 按差额增减 |
| 其他调整 | 手工修正 | 按输入金额增减 |

支持按日期、证券账户、流水类型查询，支持新增、编辑、删除。

### 3.9 多证券账户费率与手续费估算

现金设置页提供「交易费率设置」。

支持：

- 多个证券账户分别保存费率；
- 每个账户按资产类别维护佣金率、印花税率、过户费率、最低佣金；
- 交易录入选择证券账户后自动估算手续费；
- 手续费输入框可手动覆盖；
- 手动覆盖后，后续修改金额/分类/方向/账户不会自动覆盖手动值。

默认口径：

- A股权益：佣金、卖出印花税、过户费；
- ETF / REITs / 黄金：默认只按佣金估算；
- 债基：默认 0；
- 费率单位是 `%`，例如万2.5填写 `0.025`。

华泰普通账户示例：

| 分类 | 佣金率 | 印花税 | 过户费 | 最低佣金 |
|---|---:|---:|---:|---:|
| A股权益 | 0.013 | 0.05 | 0.001 | 5 |
| A股ETF / 港股ETF / REITs / 黄金 | 0.01 | 0 | 0 | 5 |
| 债基 | 0 | 0 | 0 | 0 |

### 3.10 持仓强制校正

用于解决券商实际持仓数量/成本价与系统按交易记录重算结果不一致的问题。

规则：

- 校正记录是计算锚点，不修改历史交易；
- 无校正时：所有交易记录参与持仓重算；
- 有校正时：以最新校正锚点为基础，只叠加校正日期之后的交易；
- 校正记录可查询、可删除；
- 删除校正记录后系统重新按剩余记录计算。

适用场景：

- 历史交易不完整；
- 早期手续费、分红、拆分等记录缺失；
- 券商成本口径与系统普通成本不一致；
- 需要以券商当前显示为准重新锚定。

### 3.11 普通成本与摊薄成本

系统保留两个成本口径：

```text
普通成本价 = 剩余持仓普通成本 / 剩余持仓数量
券商摊薄成本 = (累计买入现金 - 累计卖出回款 - 累计分红) / 剩余持仓数量
```

说明：

- 普通成本用于计算剩余确认持仓的浮动盈亏；
- 券商摊薄成本尽量贴近券商 APP 的持仓成本口径；
- 当历史卖出回款 + 分红超过累计买入现金时，摊薄成本可以为负数；
- 累计盈亏仍按普通成本口径计算，避免负成本导致重复计算收益。

农业银行示例：

```text
(累计买入 288,018.19 - 卖出回款 330,137.19 - 累计分红 43,934.21) / 41,400 ≈ -2.0786
```

### 3.12 近一年标的收益率

持仓明细新增「近一年收益」列。

口径：

```text
近一年标的收益率 = 标的自身过去一年价格/净值变化率
```

说明：

- 它是标的历史表现，不是账户实际持有收益；
- 不考虑你的买入/卖出时点、仓位变化、成本价、摊薄成本和资金流；
- 可与「预计年化收益」对比：预计年化是未来假设，近一年收益是过去表现；
- A股 / ETF / REIT / 黄金优先使用前复权 K 线；
- 场外基金使用天天基金累计净值，必要时回退到单位净值；
- 外部行情源缺失或失败时显示「暂无数据」。

## 4. 核心数据口径

### 4.1 总资产

```text
总资产 = 投资账户市值 + 证券现金 + 银行存款 + 申购在途
```

### 4.2 投资账户市值

```text
投资账户市值 = Σ(确认持仓数量 × 最新价)
```

### 4.3 投资账户盈亏

```text
投资账户盈亏 = Σ[(最新价 - 普通成本价) × 持仓数量 + 累计分红]
```

说明：申购在途不计入投资账户盈亏。

### 4.4 收益率

```text
收益率 = 累计盈亏 / 普通持仓成本
```

### 4.5 组合预计年化

```text
组合预计年化 = Σ(资产金额 × 预计年化收益率) / Σ资产金额
```

### 4.6 申购在途

适用场景：场外基金申购后，券商现金已扣减或冻结，但份额和确认净值尚未给出。

```text
申购在途 = Σ(方向为 申购待确认 的 金额 + 手续费)
```

规则：

- 计入总资产；
- 计入资产配置中的固收；
- 不计入正式持仓数量；
- 不计入持仓成本和盈亏；
- 份额确认后，编辑原交易为 `买入`，补充实际份额和确认净值。

## 7. 常见操作

### 7.1 记录/更新今日资产快照

1. 打开「资产快照」。
2. 点击「记录/更新今日快照」。
3. 当天没有快照时系统新建；当天已有快照时系统更新当天数据。

### 7.2 录入普通买入交易

1. 打开「交易录入/管理」。
2. 方向选择 `买入`。
3. 选择证券账户。
4. 输入代码、名称、分类。
5. 手动输入数量、单价、金额。
6. 系统自动估算手续费，如与券商成交单不一致，可手动覆盖。
7. 提交记录。

### 7.3 录入基金申购待确认

1. 打开「交易录入/管理」。
2. 方向选择 `申购待确认`。
3. 分类选择 `债基`。
4. 数量和单价可保持 `0`。
5. 金额填写实际申购金额。
6. 提交后，该金额进入「申购在途」，同时证券现金自动扣减。
7. 份额/净值确认后，编辑同一笔交易，方向改为 `买入`，补充确认份额和确认净值。

### 7.4 校准证券现金

1. 打开「现金设置」。
2. 查看当前自动余额。
3. 如发生银证转账或券商余额与系统不一致，输入券商显示的当前现金余额。
4. 点击「保存校准」。
5. 系统会自动生成一条「现金校准」流水。

### 7.5 记录银证转入/转出

1. 打开「现金设置」。
2. 在「证券资金流水」选择日期、证券账户、类型。
3. 类型选择 `银证转入` 或 `银证转出`。
4. 填写金额和备注。
5. 点击「新增流水」。
6. 系统自动更新证券现金，并在流水表中保留记录。

### 7.6 持仓强制校正

1. 打开「持仓明细」。
2. 在对应标的右侧点击「校正」。
3. 选择校正日期，录入券商实际数量、成本价、累计分红。
4. 点击保存后，系统以该记录作为新的持仓锚点并重新计算持仓。
5. 点击「记录」可查看历史校正记录，误录后可删除。

### 7.7 设置多证券账户费率

1. 打开「现金设置」。
2. 在「交易费率设置」中选择当前费率账户。
3. 如需新增券商，填写账户名称并点击「新增账户」。
4. 按类别修改佣金率、印花税、过户费和最低佣金。
5. 点击「保存费率设置」。
6. 后续交易录入选择对应证券账户后，手续费会按该账户费率自动估算。

### 7.8 导入/导出交易记录

1. 打开「交易录入/管理」。
2. 点击「下载交易模板」，按模板填写 CSV。
3. 点击「导出交易」可备份当前全部交易记录。
4. 点击「导入交易」上传 CSV。
5. 系统导入前会自动备份数据库，导入成功后自动重算持仓和证券现金。

### 7.9 导入/导出银行存款

1. 打开「银行存款」。
2. 点击「下载存款模板」，按模板填写 CSV。
3. 点击「导出存款」可备份当前存款明细。
4. 点击「导入存款」上传 CSV。
5. 系统导入前会自动备份数据库。

### 7.10 修改持仓预计年化收益

1. 打开「持仓明细」。
2. 在目标持仓右侧点击「年化」或「编辑」。
3. 输入新的预计年化收益。
4. 保存后，资产配置页的组合预计年化会重新计算。

### 7.11 同步近一年标的收益率

1. 打开「持仓明细」。
2. 点击「同步近一年收益率」。
3. 系统批量获取每个标的过去一年价格/净值数据。
4. 成功时显示百分比；失败或数据不足时显示「暂无数据」。

## 9. 注意事项

1. 本系统是本地单用户工具，不是生产级多用户系统。
2. `data/invest.db` 是核心真实数据文件，建议定期备份。
3. 交易价格不会自动填充，必须按真实成交价手动录入。
4. 分红、手续费、交易金额应以券商记录为准；系统手续费只是自动估算。
5. 最新价同步和近一年收益同步依赖外部行情源，特殊标的可能同步失败。
6. 预计年化收益是个人假设，不代表收益承诺。
7. 近一年标的收益率是历史价格/净值回溯指标，不代表账户实际收益。
8. `申购待确认` 只用于尚未确认份额/净值的申购；确认后应改为 `买入`。
9. 同日资产快照会被更新，不会新增多条同日记录。
10. 证券现金由资金流水和交易现金流共同自动联动。
11. 银证转入/转出应记录在「证券资金流水」；买入/卖出/分红/申购待确认应记录在「交易录入/管理」。
12. 多证券账户只影响交易归属和手续费估算；删除费率账户不会删除交易记录。
13. 导入 CSV 会真实写入数据；导入前系统会自动备份，但仍建议先导出一份当前数据。
14. 高风险修改前应先执行 `python3 scripts/safety_snapshot.py --label xxx`。
